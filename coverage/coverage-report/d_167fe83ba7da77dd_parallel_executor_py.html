29class ParallelExecutor(object): 

30 """ 

31 :api_attr: Static Graph 

32 

33 The ParallelExecutor is an upgraded version of :code:`paddle.static.Executor` that supports multi-node model 

34 training and testing based on the data-parallel mode. In data-parallel mode, 

35 ParallelExecutor will broadcast the parameters from Node0 to other nodes during 

36 construction and copy the input Program to other nodes from Node0 to make sure 

37 that the initial state on each node is the same. Each node runs the model independently 

38 and the parameters' gradient is aggregated between those nodes during backward 

39 computation, and then each node independently updates its parameters. If you use 

40 the GPU to run the model, i.e. use_cuda=True, the node refers to the GPU, 

41 ParallelExecutor will automatically get the GPU resources available on the 

42 current machine, users can also set the available GPU resources in the environment 

43 variable, for example: want to use GPU0, GPU1, export CUDA_VISIBLEDEVICES=0,1; 

44 If the operation is performed on the CPU, i.e. use_cuda=False, the node refers to the CPU. 

45 **Note: At this time, the user needs to manually add CPU_NUM to the environment variable 

46 and set the number of CPU devices. For example, export CPU_NUM=4, if the environment 

47 variable is not set, the executor will add the variable to the environment variable 

48 and set it to 1.** 

49 

50 

51 Args: 

52 use_cuda (bool): Whether to use CUDA or not. 

53 loss_name (str): This parameter is the name of the loss Tensor of the 

54 model. **Note: If it is data-parallel model training, you must set loss_name, 

55 otherwise, the results may be wrong**. The default is None. 

56 main_program (Program): This parameter represents the Program to be executed. 

57 If this parameter is not provided, that parameter is None, the program will 

58 be set to :code:`paddle.static.default_main_program()`. The default is None. 

59 share_vars_from(ParallelExecutor): If share_vars_from is set, the current 

60 ParallelExecutor will share the parameters with the ParallelExecutor 

61 specified by share_vars_from. This parameter needs to be set when model testing 

62 is required during model training, and the data parallel mode is used for 

63 training and testing. Since ParallelExecutor will only distribute parameter 

64 variables to other devices when it is first executed, the ParallelExecutor 

65 specified by share_vars_from must be run before the current ParallelExecutor. 

66 The default is None. 

67 exec_strategy(ExecutionStrategy): exec_strategy specifies the options that can 

68 be changed when running the current model, such as the thread pool size. 

69 For more information about exec_strategy, please refer to :code:`paddle.static.ExecutionStrategy`. 

70 The default is None. 

71 build_strategy(BuildStrategy): By configuring build_strategy, we can 

72 optimize the computational graph, such as operators' fusion in the 

73 computational graph and memory optimization during the execution 

74 of the computational graph. For more information about build_strategy, 

75 please refer to :code:`paddle.static.BuildStrategy`. The default is None. 

76 num_trainers(int): This parameter needs to be set in GPU distributed training. 

77 If the parameter value is greater than 1, NCCL will be initialized by multi-level 

78 nodes. Each node should have the same number of GPUs. The default is 1. 

79 trainer_id(int): This parameter needs to be set when performing GPU distributed 

80 training. This parameter must be used with the num_trainers parameter. 

81 Trainer_id indicates the "rank" of the current node. The trainer_id starts 

82 counting from 0. The default is 0. 

83 scope(Scope): Specifies the scope in which the program is executed. 

84 The default is paddle.static.global_scope(). 

85 

86 Returns: 

87 ParallelExecutor: The initialized ParallelExecutor object. 

88 

89 Raises: 

90 TypeError: If share_vars_from is provided, but not ParallelExecutor object. 

91 

92 NOTES: 

93 

94 1. If you only use ParallelExecutor to do multi-card test, you don't need to set loss_name 

95 and share_vars_from. 

96 

97 2. If you need to train and test the model with ParallelExecutor, the share_vars_from 

98 must be set when building the ParallelExecutor corresponding to the model test. 

99 Otherwise, the parameters used in the model test and the model training are inconsistent. 

100 

101 Examples: 

102 .. code-block:: python 

103 

104 import paddle 

105 import numpy 

106 import os 

107 

108 use_cuda = True 

109 paddle.enable_static() 

110 place = paddle.CUDAPlace(0) if use_cuda else paddle.CPUPlace() 

111 

112 # NOTE: If you use CPU to run the program, you need 

113 # to specify the CPU_NUM, otherwise, PaddlePaddle will use 

114 # all the number of the logic core as the CPU_NUM, 

115 # in that case, the batch size of the input should be 

116 # greater than CPU_NUM, if not, the process will be 

117 # failed by an exception. 

118 if not use_cuda: 

119 os.environ['CPU_NUM'] = str(2) 

120 

121 exe = paddle.static.Executor(place) 

122 

123 train_program = paddle.static.Program() 

124 startup_program = paddle.static.Program() 

125 with paddle.static.program_guard(train_program, startup_program): 

126 data = paddle.static.data(name='X', shape=[None, 1], dtype='float32') 

127 hidden = paddle.static.nn.fc(data, 10) 

128 loss = paddle.mean(hidden) 

129 test_program = paddle.static.default_main_program().clone(for_test=True) 

130 paddle.optimizer.SGD(learning_rate=0.01).minimize(loss) 

131 

132 exe.run(startup_program) 

133 

134 train_exe = paddle.static.ParallelExecutor(use_cuda=use_cuda, 

135 main_program=train_program, 

136 loss_name=loss.name) 

137 # Note: if share_vars_from is not set here, the test parameter is different to the train one 

138 test_exe = paddle.static.ParallelExecutor(use_cuda=use_cuda, 

139 main_program=test_program, 

140 share_vars_from=train_exe) 

141 

142 x = numpy.random.random(size=(10, 1)).astype('float32') 

143 loss_data, = train_exe.run(feed={"X": x}, 

144 fetch_list=[loss.name]) 

145 

146 loss_data, = test_exe.run(feed={"X": x}, 

147 fetch_list=[loss.name]) 

148 

149 """

201 def run(self, fetch_list, feed=None, feed_dict=None, return_numpy=True): 

202 """ 

203 This interface is used to run the current model. It should be noted 

204 that the executor will execute all the operators in the Program, 

205 and will not prune some operators in the Program according to the 

206 fetch_list. 

207 

208 Args: 

209 fetch_list(list): This parameter represents the Tensors that need to be returned 

210 after the model runs. The default is None. 

211 feed(list|dict): This parameter represents the input Tensors of the model. 

212 If it is single card training, the feed is dict type, and if it is multi-card 

213 training, the parameter feed can be dict or list of Tensor. If the 

214 parameter type is dict, the data in the feed will be split and sent to 

215 multiple devices (CPU/GPU), that is to say, the input data will be evenly 

216 sent to different devices, so you should make sure the number of samples of 

217 the current mini-batch must be greater than the number of places; 

218 if the parameter type is list, those data are copied directly to each device, 

219 so the length of this list should be equal to the number of places. 

220 The default is None. 

221 feed_dict: Alias for feed parameter, for backward compatibility. 

222 This parameter has been deprecated. Default None. 

223 return_numpy(bool): This parameter indicates whether convert the fetched Tensors 

224 (the Tensor specified in the fetch list) to numpy.ndarray. if it is False, 

225 the type of the return value is a list of :code:`LoDTensor`. The default is True. 

226 

227 Returns: 

228 List: The fetched result list. 

229 

230 Raises: 

231 ValueError: If the feed is a list, but its length is not equal the 

232 length of active places, or its element's is not dict. 

233 

234 NOTES: 

235 1. If the feed parameter is dict type, the input data will be evenly distributed 

236 to different cards. For example, using two GPUs to run the model, the input 

237 sample number is 3, that is, [0, 1, 2], the sample number on GPU0 is 1, 

238 that is, [0], and the sample number on GPU1 is 2, that is, [1, 2]. 

239 If the number of samples is less than the number of devices, the program will 

240 throw an exception, so when running the model, you should make sure that the 

241 number of samples of the last batch of the data set should be greater than the 

242 number of CPU cores or GPU cards, if it is less than, it is recommended that 

243 the batch be discarded. 

244 2. If the number of CPU cores or GPU cards available is greater than 1, the fetch 

245 results are spliced together in dimension 0 for the same Tensor values 

246 (Tensors in fetch_list) on different devices. 

247 

248 

249 Examples: 

250 .. code-block:: python 

251 

252 import paddle 

253 import numpy 

254 import os 

255 

256 use_cuda = True 

257 paddle.enable_static() 

258 place = paddle.CUDAPlace(0) if use_cuda else paddle.CPUPlace() 

259 

260 # NOTE: If you use CPU to run the program, you need 

261 # to specify the CPU_NUM, otherwise, PaddlePaddle will use 

262 # all the number of the logic core as the CPU_NUM, 

263 # in that case, the batch size of the input should be 

264 # greater than CPU_NUM, if not, the process will be 

265 # failed by an exception. 

266 if not use_cuda: 

267 os.environ['CPU_NUM'] = str(2) 

268 

269 exe = paddle.static.Executor(place) 

270 

271 train_program = paddle.static.Program() 

272 startup_program = paddle.static.Program() 

273 with paddle.static.program_guard(train_program, startup_program): 

274 data = paddle.static.data(name='X', shape=[None, 1], dtype='float32') 

275 hidden = paddle.static.nn.fc(data, 10) 

276 loss = paddle.mean(hidden) 

277 paddle.optimizer.SGD(learning_rate=0.01).minimize(loss) 

278 

279 exe.run(startup_program) 

280 

281 train_exe = paddle.static.ParallelExecutor(use_cuda=use_cuda, 

282 main_program=train_program, 

283 loss_name=loss.name) 

284 

285 # If the feed is a dict: 

286 # the image will be split into devices. If there is two devices 

287 # each device will process an image with shape (5, 1) 

288 x = numpy.random.random(size=(10, 1)).astype('float32') 

289 loss_data, = train_exe.run(feed={"X": x}, 

290 fetch_list=[loss.name]) 

291 

292 # If the feed is a list: 

293 # each device will process each element in the list. 

294 # the 1st device will process an image with shape (10, 1) 

295 # the 2nd device will process an image with shape (9, 1) 

296 # 

297 # you can use exe.device_count to get the device number. 

298 x2 = numpy.random.random(size=(9, 1)).astype('float32') 

299 loss_data, = train_exe.run(feed=[{"X": x}, {"X": x2}], 

300 fetch_list=[loss.name]) 

301 

302 """ 

303 return self._exe.run(program=self._compiled_program, 

304 scope=self._scope, 

305 feed=feed, 

306 fetch_list=fetch_list, 

307 return_numpy=return_numpy)

309 @property

311 return len(self._places)

313 def drop_local_exe_scopes(self): 

314 """ 

315 Drop the local execution scopes immediately. In order to avoid frequently 

316 application and release of temporary variables, the strategy adopted by 

317 ParallelExecutor is to drop the local execution scopes after several iterations. 

318 ParallelExecutor provides the num_iteration_per_drop_scope option in 

319 :code:`paddle.static.ExecutionStrategy`, which indicates how many iterations are intervened to 

320 drop the local execution scopes. If the num_iteration_per_drop_scope value 

321 is 100, but you want to drop the local execution scopes after 50 iterations, 

322 you can call the interface manually. 

323 

324 Returns: 

325 None 

326 

327 Examples: 

328 .. code-block:: python 

329 

330 import paddle 

331 import numpy 

332 import os 

333 

334 use_cuda = True 

335 # NOTE: If you use CPU to run the program, you need 

336 # to specify the CPU_NUM, otherwise, PaddlePaddle will use 

337 # all the number of the logic core as the CPU_NUM, 

338 # in that case, the batch size of the input should be 

339 # greater than CPU_NUM, if not, the process will be 

340 # failed by an exception. 

341 if not use_cuda: 

342 os.environ['CPU_NUM'] = str(2) 

343 

344 paddle.enable_static() 

345 train_program = paddle.static.Program() 

346 startup_program = paddle.static.Program() 

347 with paddle.static.program_guard(train_program, startup_program): 

348 data = paddle.static.data(name='X', shape=[None, 1], dtype='float32') 

349 hidden = paddle.static.nn.fc(data, 10) 

350 loss = paddle.mean(hidden) 

351 

352 place = paddle.CUDAPlace(0) if use_cuda else paddle.CPUPlace() 

353 exe = paddle.static.Executor(place) 

354 exe.run(startup_program) 

355 

356 parallel_exe = paddle.static.ParallelExecutor(use_cuda=use_cuda, 

357 main_program=train_program, 

358 loss_name=loss.name) 

359 

360 x = numpy.random.random(size=(10, 1)).astype('float32') 

361 loss_data, = parallel_exe.run(feed={"X": x}, 

362 fetch_list=[loss.name]) 

363 

364 parallel_exe.drop_local_exe_scopes() 

365 

366 """ 

367 check_type(self._compiled_program._executor, 

368 "the Executor of compiled program", core.ParallelExecutor, 

369 "ParallelExecutor.drop_local_exe_scopes") 

370 self._compiled_program._executor.drop_local_exe_scopes() 

371

374 check_type(self._compiled_program._executor, 

375 "the Executor of compiled program", core.ParallelExecutor,
