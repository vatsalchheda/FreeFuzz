74def retinanet_target_assign(bbox_pred, 

75 cls_logits, 

76 anchor_box, 

77 anchor_var, 

78 gt_boxes, 

79 gt_labels, 

80 is_crowd, 

81 im_info, 

82 num_classes=1, 

83 positive_overlap=0.5, 

84 negative_overlap=0.4): 

85 r""" 

86 **Target Assign Layer for the detector RetinaNet.** 

87 

88 This OP finds out positive and negative samples from all anchors 

89 for training the detector `RetinaNet <https://arxiv.org/abs/1708.02002>`_ , 

90 and assigns target labels for classification along with target locations for 

91 regression to each sample, then takes out the part belonging to positive and 

92 negative samples from category prediction( :attr:`cls_logits`) and location 

93 prediction( :attr:`bbox_pred`) which belong to all anchors. 

94 

95 The searching principles for positive and negative samples are as followed: 

96 

97 1. Anchors are assigned to ground-truth boxes when it has the highest IoU 

98 overlap with a ground-truth box. 

99 

100 2. Anchors are assigned to ground-truth boxes when it has an IoU overlap 

101 higher than :attr:`positive_overlap` with any ground-truth box. 

102 

103 3. Anchors are assigned to background when its IoU overlap is lower than 

104 :attr:`negative_overlap` for all ground-truth boxes. 

105 

106 4. Anchors which do not meet the above conditions do not participate in 

107 the training process. 

108 

109 Retinanet predicts a :math:`C`-vector for classification and a 4-vector for box 

110 regression for each anchor, hence the target label for each positive(or negative) 

111 sample is a :math:`C`-vector and the target locations for each positive sample 

112 is a 4-vector. As for a positive sample, if the category of its assigned 

113 ground-truth box is class :math:`i`, the corresponding entry in its length 

114 :math:`C` label vector is set to 1 and all other entries is set to 0, its box 

115 regression targets are computed as the offset between itself and its assigned 

116 ground-truth box. As for a negative sample, all entries in its length :math:`C` 

117 label vector are set to 0 and box regression targets are omitted because 

118 negative samples do not participate in the training process of location 

119 regression. 

120 

121 After the assignment, the part belonging to positive and negative samples is 

122 taken out from category prediction( :attr:`cls_logits` ), and the part 

123 belonging to positive samples is taken out from location 

124 prediction( :attr:`bbox_pred` ). 

125 

126 Args: 

127 bbox_pred(Variable): A 3-D Tensor with shape :math:`[N, M, 4]` represents 

128 the predicted locations of all anchors. :math:`N` is the batch size( the 

129 number of images in a mini-batch), :math:`M` is the number of all anchors 

130 of one image, and each anchor has 4 coordinate values. The data type of 

131 :attr:`bbox_pred` is float32 or float64. 

132 cls_logits(Variable): A 3-D Tensor with shape :math:`[N, M, C]` represents 

133 the predicted categories of all anchors. :math:`N` is the batch size, 

134 :math:`M` is the number of all anchors of one image, and :math:`C` is 

135 the number of categories (**Notice: excluding background**). The data type 

136 of :attr:`cls_logits` is float32 or float64. 

137 anchor_box(Variable): A 2-D Tensor with shape :math:`[M, 4]` represents 

138 the locations of all anchors. :math:`M` is the number of all anchors of 

139 one image, each anchor is represented as :math:`[xmin, ymin, xmax, ymax]`, 

140 :math:`[xmin, ymin]` is the left top coordinate of the anchor box, 

141 :math:`[xmax, ymax]` is the right bottom coordinate of the anchor box. 

142 The data type of :attr:`anchor_box` is float32 or float64. Please refer 

143 to the OP :ref:`api_fluid_layers_anchor_generator`  

144 for the generation of :attr:`anchor_box`. 

145 anchor_var(Variable): A 2-D Tensor with shape :math:`[M,4]` represents the expanded  

146 factors of anchor locations used in loss function. :math:`M` is number of 

147 all anchors of one image, each anchor possesses a 4-vector expanded factor. 

148 The data type of :attr:`anchor_var` is float32 or float64. Please refer 

149 to the OP :ref:`api_fluid_layers_anchor_generator` 

150 for the generation of :attr:`anchor_var`. 

151 gt_boxes(Variable): A 1-level 2-D LoDTensor with shape :math:`[G, 4]` represents 

152 locations of all ground-truth boxes. :math:`G` is the total number of 

153 all ground-truth boxes in a mini-batch, and each ground-truth box has 4 

154 coordinate values. The data type of :attr:`gt_boxes` is float32 or 

155 float64. 

156 gt_labels(variable): A 1-level 2-D LoDTensor with shape :math:`[G, 1]` represents 

157 categories of all ground-truth boxes, and the values are in the range of 

158 :math:`[1, C]`. :math:`G` is the total number of all ground-truth boxes 

159 in a mini-batch, and each ground-truth box has one category. The data type 

160 of :attr:`gt_labels` is int32. 

161 is_crowd(Variable): A 1-level 1-D LoDTensor with shape :math:`[G]` which 

162 indicates whether a ground-truth box is a crowd. If the value is 1, the 

163 corresponding box is a crowd, it is ignored during training. :math:`G` is 

164 the total number of all ground-truth boxes in a mini-batch. The data type 

165 of :attr:`is_crowd` is int32. 

166 im_info(Variable): A 2-D Tensor with shape [N, 3] represents the size 

167 information of input images. :math:`N` is the batch size, the size 

168 information of each image is a 3-vector which are the height and width 

169 of the network input along with the factor scaling the origin image to 

170 the network input. The data type of :attr:`im_info` is float32. 

171 num_classes(int32): The number of categories for classification, the default 

172 value is 1. 

173 positive_overlap(float32): Minimum overlap required between an anchor 

174 and ground-truth box for the anchor to be a positive sample, the default 

175 value is 0.5. 

176 negative_overlap(float32): Maximum overlap allowed between an anchor 

177 and ground-truth box for the anchor to be a negative sample, the default 

178 value is 0.4. :attr:`negative_overlap` should be less than or equal to 

179 :attr:`positive_overlap`, if not, the actual value of 

180 :attr:`positive_overlap` is :attr:`negative_overlap`. 

181 

182 Returns: 

183 A tuple with 6 Variables: 

184  

185 **predict_scores** (Variable): A 2-D Tensor with shape :math:`[F+B, C]` represents 

186 category prediction belonging to positive and negative samples. :math:`F` 

187 is the number of positive samples in a mini-batch, :math:`B` is the number 

188 of negative samples, and :math:`C` is the number of categories 

189 (**Notice: excluding background**). The data type of :attr:`predict_scores` 

190 is float32 or float64. 

191 

192 **predict_location** (Variable): A 2-D Tensor with shape :math:`[F, 4]` represents 

193 location prediction belonging to positive samples. :math:`F` is the number 

194 of positive samples. :math:`F` is the number of positive samples, and each 

195 sample has 4 coordinate values. The data type of :attr:`predict_location` 

196 is float32 or float64. 

197 

198 **target_label** (Variable): A 2-D Tensor with shape :math:`[F+B, 1]` represents 

199 target labels for classification belonging to positive and negative 

200 samples. :math:`F` is the number of positive samples, :math:`B` is the 

201 number of negative, and each sample has one target category. The data type 

202 of :attr:`target_label` is int32. 

203 

204 **target_bbox** (Variable): A 2-D Tensor with shape :math:`[F, 4]` represents 

205 target locations for box regression belonging to positive samples. 

206 :math:`F` is the number of positive samples, and each sample has 4 

207 coordinate values. The data type of :attr:`target_bbox` is float32 or 

208 float64. 

209 

210 **bbox_inside_weight** (Variable): A 2-D Tensor with shape :math:`[F, 4]` 

211 represents whether a positive sample is fake positive, if a positive 

212 sample is false positive, the corresponding entries in 

213 :attr:`bbox_inside_weight` are set 0, otherwise 1. :math:`F` is the number 

214 of total positive samples in a mini-batch, and each sample has 4 

215 coordinate values. The data type of :attr:`bbox_inside_weight` is float32 

216 or float64. 

217 

218 **fg_num** (Variable): A 2-D Tensor with shape :math:`[N, 1]` represents the number 

219 of positive samples. :math:`N` is the batch size. **Notice: The number 

220 of positive samples is used as the denominator of later loss function, 

221 to avoid the condition that the denominator is zero, this OP has added 1 

222 to the actual number of positive samples of each image.** The data type of 

223 :attr:`fg_num` is int32. 

224 

225 Examples: 

226 .. code-block:: python 

227 

228 import paddle.fluid as fluid 

229 bbox_pred = fluid.data(name='bbox_pred', shape=[1, 100, 4], 

230 dtype='float32') 

231 cls_logits = fluid.data(name='cls_logits', shape=[1, 100, 10], 

232 dtype='float32') 

233 anchor_box = fluid.data(name='anchor_box', shape=[100, 4], 

234 dtype='float32') 

235 anchor_var = fluid.data(name='anchor_var', shape=[100, 4], 

236 dtype='float32') 

237 gt_boxes = fluid.data(name='gt_boxes', shape=[10, 4], 

238 dtype='float32') 

239 gt_labels = fluid.data(name='gt_labels', shape=[10, 1], 

240 dtype='int32') 

241 is_crowd = fluid.data(name='is_crowd', shape=[1], 

242 dtype='int32') 

243 im_info = fluid.data(name='im_info', shape=[1, 3], 

244 dtype='float32') 

245 score_pred, loc_pred, score_target, loc_target, bbox_inside_weight, fg_num = \\ 

246 fluid.layers.retinanet_target_assign(bbox_pred, cls_logits, anchor_box, 

247 anchor_var, gt_boxes, gt_labels, is_crowd, im_info, 10) 

248 

249 """ 

250 

251 check_variable_and_dtype(bbox_pred, 'bbox_pred', ['float32', 'float64'], 

252 'retinanet_target_assign') 

253 check_variable_and_dtype(cls_logits, 'cls_logits', ['float32', 'float64'], 

254 'retinanet_target_assign') 

255 check_variable_and_dtype(anchor_box, 'anchor_box', ['float32', 'float64'], 

256 'retinanet_target_assign') 

257 check_variable_and_dtype(anchor_var, 'anchor_var', ['float32', 'float64'], 

258 'retinanet_target_assign') 

259 check_variable_and_dtype(gt_boxes, 'gt_boxes', ['float32', 'float64'], 

260 'retinanet_target_assign') 

261 check_variable_and_dtype(gt_labels, 'gt_labels', ['int32'], 

262 'retinanet_target_assign') 

263 check_variable_and_dtype(is_crowd, 'is_crowd', ['int32'], 

264 'retinanet_target_assign') 

265 check_variable_and_dtype(im_info, 'im_info', ['float32', 'float64'], 

266 'retinanet_target_assign') 

267

269 # Assign target label to anchors 

270 loc_index = helper.create_variable_for_type_inference(dtype='int32') 

271 score_index = helper.create_variable_for_type_inference(dtype='int32') 

272 target_label = helper.create_variable_for_type_inference(dtype='int32') 

273 target_bbox = helper.create_variable_for_type_inference( 

274 dtype=anchor_box.dtype) 

275 bbox_inside_weight = helper.create_variable_for_type_inference( 

276 dtype=anchor_box.dtype) 

277 fg_num = helper.create_variable_for_type_inference(dtype='int32') 

278 helper.append_op(type="retinanet_target_assign", 

279 inputs={ 

280 'Anchor': anchor_box, 

281 'GtBoxes': gt_boxes, 

282 'GtLabels': gt_labels, 

283 'IsCrowd': is_crowd, 

284 'ImInfo': im_info 

285 }, 

286 outputs={ 

287 'LocationIndex': loc_index, 

288 'ScoreIndex': score_index, 

289 'TargetLabel': target_label, 

290 'TargetBBox': target_bbox, 

291 'BBoxInsideWeight': bbox_inside_weight, 

292 'ForegroundNumber': fg_num 

293 }, 

294 attrs={ 

295 'positive_overlap': positive_overlap, 

296 'negative_overlap': negative_overlap 

297 }) 

298 

299 loc_index.stop_gradient = True 

300 score_index.stop_gradient = True 

301 target_label.stop_gradient = True 

302 target_bbox.stop_gradient = True 

303 bbox_inside_weight.stop_gradient = True 

304 fg_num.stop_gradient = True 

305 

306 cls_logits = nn.reshape(x=cls_logits, shape=(-1, num_classes)) 

307 bbox_pred = nn.reshape(x=bbox_pred, shape=(-1, 4)) 

308 predicted_cls_logits = nn.gather(cls_logits, score_index) 

309 predicted_bbox_pred = nn.gather(bbox_pred, loc_index) 

310 

311 return predicted_cls_logits, predicted_bbox_pred, target_label, target_bbox, bbox_inside_weight, fg_num

314def rpn_target_assign(bbox_pred, 

315 cls_logits, 

316 anchor_box, 

317 anchor_var, 

318 gt_boxes, 

319 is_crowd, 

320 im_info, 

321 rpn_batch_size_per_im=256, 

322 rpn_straddle_thresh=0.0, 

323 rpn_fg_fraction=0.5, 

324 rpn_positive_overlap=0.7, 

325 rpn_negative_overlap=0.3, 

326 use_random=True): 

327 """ 

328 **Target Assign Layer for region proposal network (RPN) in Faster-RCNN detection.** 

329 

330 This layer can be, for given the Intersection-over-Union (IoU) overlap 

331 between anchors and ground truth boxes, to assign classification and 

332 regression targets to each each anchor, these target labels are used for 

333 train RPN. The classification targets is a binary class label (of being 

334 an object or not). Following the paper of Faster-RCNN, the positive labels 

335 are two kinds of anchors: (i) the anchor/anchors with the highest IoU 

336 overlap with a ground-truth box, or (ii) an anchor that has an IoU overlap 

337 higher than rpn_positive_overlap(0.7) with any ground-truth box. Note 

338 that a single ground-truth box may assign positive labels to multiple 

339 anchors. A non-positive anchor is when its IoU ratio is lower than 

340 rpn_negative_overlap (0.3) for all ground-truth boxes. Anchors that are 

341 neither positive nor negative do not contribute to the training objective. 

342 The regression targets are the encoded ground-truth boxes associated with 

343 the positive anchors. 

344 

345 Args: 

346 bbox_pred(Variable): A 3-D Tensor with shape [N, M, 4] represents the 

347 predicted locations of M bounding bboxes. N is the batch size, 

348 and each bounding box has four coordinate values and the layout 

349 is [xmin, ymin, xmax, ymax]. The data type can be float32 or float64. 

350 cls_logits(Variable): A 3-D Tensor with shape [N, M, 1] represents the 

351 predicted confidence predictions. N is the batch size, 1 is the 

352 frontground and background sigmoid, M is number of bounding boxes. 

353 The data type can be float32 or float64. 

354 anchor_box(Variable): A 2-D Tensor with shape [M, 4] holds M boxes, 

355 each box is represented as [xmin, ymin, xmax, ymax], 

356 [xmin, ymin] is the left top coordinate of the anchor box, 

357 if the input is image feature map, they are close to the origin 

358 of the coordinate system. [xmax, ymax] is the right bottom 

359 coordinate of the anchor box. The data type can be float32 or float64. 

360 anchor_var(Variable): A 2-D Tensor with shape [M,4] holds expanded  

361 variances of anchors. The data type can be float32 or float64. 

362 gt_boxes (Variable): The ground-truth bounding boxes (bboxes) are a 2D 

363 LoDTensor with shape [Ng, 4], Ng is the total number of ground-truth 

364 bboxes of mini-batch input. The data type can be float32 or float64. 

365 is_crowd (Variable): A 1-D LoDTensor which indicates groud-truth is crowd. 

366 The data type must be int32. 

367 im_info (Variable): A 2-D LoDTensor with shape [N, 3]. N is the batch size, 

368 3 is the height, width and scale. 

369 rpn_batch_size_per_im(int): Total number of RPN examples per image. 

370 The data type must be int32. 

371 rpn_straddle_thresh(float): Remove RPN anchors that go outside the image 

372 by straddle_thresh pixels. The data type must be float32. 

373 rpn_fg_fraction(float): Target fraction of RoI minibatch that is labeled 

374 foreground (i.e. class > 0), 0-th class is background. The data type must be float32. 

375 rpn_positive_overlap(float): Minimum overlap required between an anchor 

376 and ground-truth box for the (anchor, gt box) pair to be a positive 

377 example. The data type must be float32. 

378 rpn_negative_overlap(float): Maximum overlap allowed between an anchor 

379 and ground-truth box for the (anchor, gt box) pair to be a negative 

380 examples. The data type must be float32. 

381 

382 Returns: 

383 tuple: 

384 A tuple(predicted_scores, predicted_location, target_label, 

385 target_bbox, bbox_inside_weight) is returned. The predicted_scores  

386 and predicted_location is the predicted result of the RPN. 

387 The target_label and target_bbox is the ground truth, 

388 respectively. The predicted_location is a 2D Tensor with shape 

389 [F, 4], and the shape of target_bbox is same as the shape of 

390 the predicted_location, F is the number of the foreground 

391 anchors. The predicted_scores is a 2D Tensor with shape 

392 [F + B, 1], and the shape of target_label is same as the shape 

393 of the predicted_scores, B is the number of the background 

394 anchors, the F and B is depends on the input of this operator. 

395 Bbox_inside_weight represents whether the predicted loc is fake_fg 

396 or not and the shape is [F, 4]. 

397 

398 Examples: 

399 .. code-block:: python 

400 

401 import paddle.fluid as fluid 

402 bbox_pred = fluid.data(name='bbox_pred', shape=[None, 4], dtype='float32') 

403 cls_logits = fluid.data(name='cls_logits', shape=[None, 1], dtype='float32') 

404 anchor_box = fluid.data(name='anchor_box', shape=[None, 4], dtype='float32') 

405 anchor_var = fluid.data(name='anchor_var', shape=[None, 4], dtype='float32') 

406 gt_boxes = fluid.data(name='gt_boxes', shape=[None, 4], dtype='float32') 

407 is_crowd = fluid.data(name='is_crowd', shape=[None], dtype='float32') 

408 im_info = fluid.data(name='im_infoss', shape=[None, 3], dtype='float32') 

409 loc, score, loc_target, score_target, inside_weight = fluid.layers.rpn_target_assign( 

410 bbox_pred, cls_logits, anchor_box, anchor_var, gt_boxes, is_crowd, im_info) 

411 

412 """ 

413 

414 helper = LayerHelper('rpn_target_assign', **locals()) 

415 

416 check_variable_and_dtype(bbox_pred, 'bbox_pred', ['float32', 'float64'], 

417 'rpn_target_assign') 

418 check_variable_and_dtype(cls_logits, 'cls_logits', ['float32', 'float64'], 

419 'rpn_target_assign') 

420 check_variable_and_dtype(anchor_box, 'anchor_box', ['float32', 'float64'], 

421 'rpn_target_assign') 

422 check_variable_and_dtype(anchor_var, 'anchor_var', ['float32', 'float64'], 

423 'rpn_target_assign') 

424 check_variable_and_dtype(gt_boxes, 'gt_boxes', ['float32', 'float64'], 

425 'rpn_target_assign') 

426 check_variable_and_dtype(is_crowd, 'is_crowd', ['int32'], 

427 'rpn_target_assign') 

428 check_variable_and_dtype(im_info, 'im_info', ['float32', 'float64'], 

429 'rpn_target_assign') 

430 

431 # Assign target label to anchors 

432 loc_index = helper.create_variable_for_type_inference(dtype='int32') 

433 score_index = helper.create_variable_for_type_inference(dtype='int32') 

434 target_label = helper.create_variable_for_type_inference(dtype='int32') 

435 target_bbox = helper.create_variable_for_type_inference( 

436 dtype=anchor_box.dtype) 

437 bbox_inside_weight = helper.create_variable_for_type_inference( 

438 dtype=anchor_box.dtype) 

439 helper.append_op(type="rpn_target_assign", 

440 inputs={ 

441 'Anchor': anchor_box, 

442 'GtBoxes': gt_boxes, 

443 'IsCrowd': is_crowd, 

444 'ImInfo': im_info 

445 }, 

446 outputs={ 

447 'LocationIndex': loc_index, 

448 'ScoreIndex': score_index, 

449 'TargetLabel': target_label, 

450 'TargetBBox': target_bbox, 

451 'BBoxInsideWeight': bbox_inside_weight 

452 }, 

453 attrs={ 

454 'rpn_batch_size_per_im': rpn_batch_size_per_im, 

455 'rpn_straddle_thresh': rpn_straddle_thresh, 

456 'rpn_positive_overlap': rpn_positive_overlap, 

457 'rpn_negative_overlap': rpn_negative_overlap, 

458 'rpn_fg_fraction': rpn_fg_fraction, 

459 'use_random': use_random 

460 }) 

461 

462 loc_index.stop_gradient = True 

463 score_index.stop_gradient = True 

464 target_label.stop_gradient = True 

465 target_bbox.stop_gradient = True 

466 bbox_inside_weight.stop_gradient = True 

467 

468 cls_logits = nn.reshape(x=cls_logits, shape=(-1, 1)) 

469 bbox_pred = nn.reshape(x=bbox_pred, shape=(-1, 4)) 

470 predicted_cls_logits = nn.gather(cls_logits, score_index) 

471 predicted_bbox_pred = nn.gather(bbox_pred, loc_index) 

472 

473 return predicted_cls_logits, predicted_bbox_pred, target_label, target_bbox, bbox_inside_weight

476def sigmoid_focal_loss(x, label, fg_num, gamma=2.0, alpha=0.25): 

477 r""" 

478 :alias_main: paddle.nn.functional.sigmoid_focal_loss 

479 :alias: paddle.nn.functional.sigmoid_focal_loss,paddle.nn.functional.loss.sigmoid_focal_loss 

480 :old_api: paddle.fluid.layers.sigmoid_focal_loss 

481 

482 **Sigmoid Focal Loss Operator.** 

483 

484 `Focal Loss <https://arxiv.org/abs/1708.02002>`_ is used to address the foreground-background 

485 class imbalance existed on the training phase of many computer vision tasks. This OP computes 

486 the sigmoid value for each element in the input tensor :attr:`x`, after which focal loss is 

487 measured between the sigmoid value and target label.  

488 

489 The focal loss is given as followed: 

490 

491 .. math:: 

492  

493 \\mathop{loss_{i,\\,j}}\\limits_{i\\in\\mathbb{[0,\\,N-1]},\\,j\\in\\mathbb{[0,\\,C-1]}}=\\left\\{ 

494 \\begin{array}{rcl} 

495 - \\frac{1}{fg\_num} * \\alpha * {(1 - \\sigma(x_{i,\\,j}))}^{\\gamma} * \\log(\\sigma(x_{i,\\,j})) & & {(j +1) = label_{i,\\,0}} \\\\ 

496 - \\frac{1}{fg\_num} * (1 - \\alpha) * {\sigma(x_{i,\\,j})}^{ \\gamma} * \\log(1 - \\sigma(x_{i,\\,j})) & & {(j +1)!= label_{i,\\,0}} 

497 \\end{array} \\right. 

498 

499 

500 We know that 

501  

502 .. math:: 

503 \\sigma(x_j) = \\frac{1}{1 + \\exp(-x_j)} 

504 

505 

506 Args: 

507 x(Variable): A 2-D tensor with shape :math:`[N, C]` represents the predicted categories of 

508 all samples. :math:`N` is the number of all samples responsible for optimization in 

509 a mini-batch, for example, samples are anchor boxes for object detection and :math:`N` 

510 is the total number of positive and negative samples in a mini-batch; Samples are images 

511 for image classification and :math:`N` is the number of images in a mini-batch. :math:`C` 

512 is the number of classes (**Notice: excluding background**). The data type of :attr:`x` is 

513 float32 or float64. 

514 label(Variable): A 2-D tensor with shape :math:`[N, 1]` represents the target labels for 

515 classification. :math:`N` is the number of all samples responsible for optimization in a 

516 mini-batch, each sample has one target category. The values for positive samples are in the 

517 range of :math:`[1, C]`, and the values for negative samples are 0. The data type of :attr:`label` 

518 is int32. 

519 fg_num(Variable): A 1-D tensor with shape [1] represents the number of positive samples in a 

520 mini-batch, which should be obtained before this OP. The data type of :attr:`fg_num` is int32. 

521 gamma(int|float): Hyper-parameter to balance the easy and hard examples. Default value is 

522 set to 2.0. 

523 alpha(int|float): Hyper-parameter to balance the positive and negative example. Default value 

524 is set to 0.25. 

525 

526 Returns: 

527 Variable(the data type is float32 or float64):  

528 A 2-D tensor with shape :math:`[N, C]`, which is the focal loss of each element in the input 

529 tensor :attr:`x`.

532 .. code-block:: python 

533

552 def get_focal_loss(pred, label, fg_num, num_classes): 

553 pred = fluid.layers.reshape(pred, [-1, num_classes]) 

554 label = fluid.layers.reshape(label, [-1, 1])

676 Default: 0.01.

712 loc = fluid.data(name='target_box', shape=[2, 21, 4], dtype='float32') 

713 scores = fluid.data(name='scores', shape=[2, 21, 10], dtype='float32') 

714 nmsed_outs, index = fluid.layers.detection_output(scores=scores, 

715 loc=loc, 

716 prior_box=pb, 

717 prior_box_var=pbv, 

718 return_index=True) 

719 """

722 prior_box_var=prior_box_var, 

723 target_box=loc, 

724 code_type='decode_center_size') 

725 scores = nn.softmax(input=scores) 

726 scores = nn.transpose(scores, perm=[0, 2, 1]) 

727 scores.stop_gradient = True 

728 nmsed_outs = helper.create_variable_for_type_inference( 

729 dtype=decoded_box.dtype) 

730 if return_index: 

731 index = helper.create_variable_for_type_inference(dtype='int') 

732 helper.append_op(type="multiclass_nms2", 

733 inputs={ 

734 'Scores': scores, 

735 'BBoxes': decoded_box 

736 }, 

737 outputs={ 

738 'Out': nmsed_outs, 

739 'Index': index 

740 }, 

741 attrs={ 

742 'background_label': 0, 

743 'nms_threshold': nms_threshold, 

744 'nms_top_k': nms_top_k, 

745 'keep_top_k': keep_top_k, 

746 'score_threshold': score_threshold, 

747 'nms_eta': 1.0,

754 'BBoxes': decoded_box

826@templatedoc() 

827def box_coder(prior_box, 

828 prior_box_var, 

829 target_box, 

830 code_type="encode_center_size", 

831 box_normalized=True, 

832 name=None, 

833 axis=0): 

834 r""" 

835 

836 **Box Coder Layer** 

837 

838 Encode/Decode the target bounding box with the priorbox information. 

839 

840 The Encoding schema described below: 

841 

842 .. math:: 

843 

844 ox = (tx - px) / pw / pxv 

845 

846 oy = (ty - py) / ph / pyv 

847 

848 ow = \log(\abs(tw / pw)) / pwv 

849 

850 oh = \log(\abs(th / ph)) / phv 

851 

852 The Decoding schema described below: 

853 

854 .. math:: 

855 

856 ox = (pw * pxv * tx * + px) - tw / 2 

857 

858 oy = (ph * pyv * ty * + py) - th / 2 

859 

860 ow = \exp(pwv * tw) * pw + tw / 2 

861 

862 oh = \exp(phv * th) * ph + th / 2 

863 

864 where `tx`, `ty`, `tw`, `th` denote the target box's center coordinates, 

865 width and height respectively. Similarly, `px`, `py`, `pw`, `ph` denote 

866 the priorbox's (anchor) center coordinates, width and height. `pxv`, 

867 `pyv`, `pwv`, `phv` denote the variance of the priorbox and `ox`, `oy`, 

868 `ow`, `oh` denote the encoded/decoded coordinates, width and height. 

869 

870 During Box Decoding, two modes for broadcast are supported. Say target 

871 box has shape [N, M, 4], and the shape of prior box can be [N, 4] or 

872 [M, 4]. Then prior box will broadcast to target box along the 

873 assigned axis. 

874 

875 Args: 

876 prior_box(Variable): Box list prior_box is a 2-D Tensor with shape 

877 [M, 4] holds M boxes and data type is float32 or float64. Each box 

878 is represented as [xmin, ymin, xmax, ymax], [xmin, ymin] is the 

879 left top coordinate of the anchor box, if the input is image feature 

880 map, they are close to the origin of the coordinate system. 

881 [xmax, ymax] is the right bottom coordinate of the anchor box. 

882 prior_box_var(List|Variable|None): prior_box_var supports three types 

883 of input. One is variable with shape [M, 4] which holds M group and 

884 data type is float32 or float64. The second is list consist of 

885 4 elements shared by all boxes and data type is float32 or float64. 

886 Other is None and not involved in calculation. 

887 target_box(Variable): This input can be a 2-D LoDTensor with shape 

888 [N, 4] when code_type is 'encode_center_size'. This input also can 

889 be a 3-D Tensor with shape [N, M, 4] when code_type is 

890 'decode_center_size'. Each box is represented as 

891 [xmin, ymin, xmax, ymax]. The data type is float32 or float64. 

892 This tensor can contain LoD information to represent a batch of inputs. 

893 code_type(str): The code type used with the target box. It can be 

894 `encode_center_size` or `decode_center_size`. `encode_center_size` 

895 by default. 

896 box_normalized(bool): Whether treat the priorbox as a normalized box. 

897 Set true by default. 

898 name(str, optional): For detailed information, please refer 

899 to :ref:`api_guide_Name`. Usually name is no need to set and 

900 None by default. 

901 axis(int): Which axis in PriorBox to broadcast for box decode, 

902 for example, if axis is 0 and TargetBox has shape [N, M, 4] and 

903 PriorBox has shape [M, 4], then PriorBox will broadcast to [N, M, 4] 

904 for decoding. It is only valid when code type is 

905 `decode_center_size`. Set 0 by default. 

906 

907 Returns: 

908 Variable: 

909 

910 output_box(Variable): When code_type is 'encode_center_size', the 

911 output tensor of box_coder_op with shape [N, M, 4] representing the 

912 result of N target boxes encoded with M Prior boxes and variances. 

913 When code_type is 'decode_center_size', N represents the batch size 

914 and M represents the number of decoded boxes. 

915 

916 Examples: 

917 

918 .. code-block:: python 

919 

920 import paddle.fluid as fluid 

921 import paddle 

922 paddle.enable_static() 

923 # For encode 

924 prior_box_encode = fluid.data(name='prior_box_encode', 

925 shape=[512, 4], 

926 dtype='float32') 

927 target_box_encode = fluid.data(name='target_box_encode', 

928 shape=[81, 4], 

929 dtype='float32') 

930 output_encode = fluid.layers.box_coder(prior_box=prior_box_encode, 

931 prior_box_var=[0.1,0.1,0.2,0.2], 

932 target_box=target_box_encode, 

933 code_type="encode_center_size") 

934 # For decode 

935 prior_box_decode = fluid.data(name='prior_box_decode', 

936 shape=[512, 4], 

937 dtype='float32') 

938 target_box_decode = fluid.data(name='target_box_decode', 

939 shape=[512, 81, 4], 

940 dtype='float32') 

941 output_decode = fluid.layers.box_coder(prior_box=prior_box_decode, 

942 prior_box_var=[0.1,0.1,0.2,0.2], 

943 target_box=target_box_decode, 

944 code_type="decode_center_size", 

945 box_normalized=False, 

946 axis=1) 

947 """

1316def bipartite_match(dist_matrix, 

1317 match_type=None, 

1318 dist_threshold=None, 

1319 name=None): 

1320 """ 

1321 

1322 This operator implements a greedy bipartite matching algorithm, which is 

1323 used to obtain the matching with the maximum distance based on the input 

1324 distance matrix. For input 2D matrix, the bipartite matching algorithm can 

1325 find the matched column for each row (matched means the largest distance), 

1326 also can find the matched row for each column. And this operator only 

1327 calculate matched indices from column to row. For each instance, 

1328 the number of matched indices is the column number of the input distance 

1329 matrix. **The OP only supports CPU**. 

1330 

1331 There are two outputs, matched indices and distance. 

1332 A simple description, this algorithm matched the best (maximum distance) 

1333 row entity to the column entity and the matched indices are not duplicated 

1334 in each row of ColToRowMatchIndices. If the column entity is not matched 

1335 any row entity, set -1 in ColToRowMatchIndices. 

1336 

1337 NOTE: the input DistMat can be LoDTensor (with LoD) or Tensor. 

1338 If LoDTensor with LoD, the height of ColToRowMatchIndices is batch size. 

1339 If Tensor, the height of ColToRowMatchIndices is 1. 

1340 

1341 NOTE: This API is a very low level API. It is used by :code:`ssd_loss` 

1342 layer. Please consider to use :code:`ssd_loss` instead. 

1343 

1344 Args: 

1345 dist_matrix(Variable): This input is a 2-D LoDTensor with shape 

1346 [K, M]. The data type is float32 or float64. It is pair-wise  

1347 distance matrix between the entities represented by each row and  

1348 each column. For example, assumed one entity is A with shape [K],  

1349 another entity is B with shape [M]. The dist_matrix[i][j] is the  

1350 distance between A[i] and B[j]. The bigger the distance is, the  

1351 better matching the pairs are. NOTE: This tensor can contain LoD  

1352 information to represent a batch of inputs. One instance of this  

1353 batch can contain different numbers of entities. 

1354 match_type(str, optional): The type of matching method, should be 

1355 'bipartite' or 'per_prediction'. None ('bipartite') by default. 

1356 dist_threshold(float32, optional): If `match_type` is 'per_prediction', 

1357 this threshold is to determine the extra matching bboxes based 

1358 on the maximum distance, 0.5 by default. 

1359 name(str, optional): For detailed information, please refer  

1360 to :ref:`api_guide_Name`. Usually name is no need to set and  

1361 None by default. 

1362  

1363 Returns: 

1364 Tuple: 

1365 

1366 matched_indices(Variable): A 2-D Tensor with shape [N, M]. The data 

1367 type is int32. N is the batch size. If match_indices[i][j] is -1, it 

1368 means B[j] does not match any entity in i-th instance. 

1369 Otherwise, it means B[j] is matched to row 

1370 match_indices[i][j] in i-th instance. The row number of 

1371 i-th instance is saved in match_indices[i][j]. 

1372 

1373 matched_distance(Variable): A 2-D Tensor with shape [N, M]. The data 

1374 type is float32. N is batch size. If match_indices[i][j] is -1, 

1375 match_distance[i][j] is also -1.0. Otherwise, assumed 

1376 match_distance[i][j] = d, and the row offsets of each instance 

1377 are called LoD. Then match_distance[i][j] = 

1378 dist_matrix[d+LoD[i]][j]. 

1379 

1380 Examples: 

1381 

1382 >>> import paddle.fluid as fluid 

1383 >>> x = fluid.data(name='x', shape=[None, 4], dtype='float32') 

1384 >>> y = fluid.data(name='y', shape=[None, 4], dtype='float32') 

1385 >>> iou = fluid.layers.iou_similarity(x=x, y=y) 

1386 >>> matched_indices, matched_dist = fluid.layers.bipartite_match(iou) 

1387 """ 

1388 helper = LayerHelper('bipartite_match', **locals()) 

1389 match_indices = helper.create_variable_for_type_inference(dtype='int32') 

1390 match_distance = helper.create_variable_for_type_inference( 

1391 dtype=dist_matrix.dtype) 

1392 helper.append_op(type='bipartite_match', 

1393 inputs={'DistMat': dist_matrix}, 

1394 attrs={ 

1395 'match_type': match_type, 

1396 'dist_threshold': dist_threshold, 

1397 }, 

1398 outputs={ 

1399 'ColToRowMatchIndices': match_indices, 

1400 'ColToRowMatchDist': match_distance 

1401 }) 

1402 return match_indices, match_distance

1528 r"""

1530 :alias: paddle.nn.functional.ssd_loss,paddle.nn.functional.loss.ssd_loss 

1531 :old_api: paddle.fluid.layers.ssd_loss 

1532 

1533 **Multi-box loss layer for object detection algorithm of SSD** 

1534 

1535 This layer is to compute detection loss for SSD given the location offset 

1536 predictions, confidence predictions, prior boxes and ground-truth bounding 

1537 boxes and labels, and the type of hard example mining. The returned loss 

1538 is a weighted sum of the localization loss (or regression loss) and 

1539 confidence loss (or classification loss) by performing the following steps: 

1540 

1541 1. Find matched bounding box by bipartite matching algorithm. 

1542 

1543 1.1 Compute IOU similarity between ground-truth boxes and prior boxes. 

1544 

1545 1.2 Compute matched bounding box by bipartite matching algorithm. 

1546 

1547 2. Compute confidence for mining hard examples 

1548 

1549 2.1. Get the target label based on matched indices. 

1550 

1551 2.2. Compute confidence loss. 

1552 

1553 3. Apply hard example mining to get the negative example indices and update 

1554 the matched indices. 

1555 

1556 4. Assign classification and regression targets 

1557 

1558 4.1. Encoded bbox according to the prior boxes. 

1559 

1560 4.2. Assign regression targets. 

1561 

1562 4.3. Assign classification targets. 

1563 

1564 5. Compute the overall objective loss. 

1565 

1566 5.1 Compute confidence loss. 

1567 

1568 5.2 Compute localization loss. 

1569 

1570 5.3 Compute the overall weighted loss. 

1571

1577 float64.

1580 `location`, C is the class number.The data type is float32 or 

1581 float64.

1583 LoDTensor with shape [Ng, 4], Ng is the total number of ground-truth 

1584 bboxes of mini-batch input.The data type is float32 or float64. 

1585 gt_label (Variable): The ground-truth labels are a 2D LoDTensor 

1586 with shape [Ng, 1].Ng is the total number of ground-truth bboxes of 

1587 mini-batch input, 1 is the number of class. The data type is float32 

1588 or float64. 

1589 prior_box (Variable): The prior boxes are a 2D Tensor with shape [Np, 4]. 

1590 Np and 4 are the same as they are in `location`. The data type is 

1591 float32 or float64. 

1592 prior_box_var (Variable): The variance of prior boxes are a 2D Tensor 

1593 with shape [Np, 4]. Np and 4 are the same as they are in `prior_box` 

1594 background_label (int): The index of background label, 0 by default. 

1595 overlap_threshold (float): If match_type is 'per_prediction', use 

1596 'overlap_threshold' to determine the extra matching bboxes when finding \ 

1597 matched boxes. 0.5 by default. 

1598 neg_pos_ratio (float): The ratio of the negative boxes to the positive 

1599 boxes, used only when mining_type is 'max_negative', 3.0 by default. 

1600 neg_overlap (float): The negative overlap upper bound for the unmatched 

1601 predictions. Use only when mining_type is 'max_negative', 

1602 0.5 by default. 

1603 loc_loss_weight (float): Weight for localization loss, 1.0 by default. 

1604 conf_loss_weight (float): Weight for confidence loss, 1.0 by default. 

1605 match_type (str): The type of matching method during training, should 

1606 be 'bipartite' or 'per_prediction', 'per_prediction' by default.

1652 def __reshape_to_2d(var): 

1653 return nn.flatten(x=var, axis=2)

1656 # 1.1 Compute IOU similarity between ground-truth boxes and prior boxes. 

1657 iou = iou_similarity(x=gt_box, y=prior_box) 

1658 # 1.2 Compute matched bounding box by bipartite matching algorithm. 

1659 matched_indices, matched_dist = bipartite_match(iou, match_type,

1663 # 2.1. Get the target label based on matched indices

1665 shape=(len(gt_label.shape) - 1) * (0, ) + (-1, 1))

1668 matched_indices, 

1669 mismatch_value=background_label)

1673 target_label = tensor.cast(x=target_label, dtype='int64') 

1674 target_label = __reshape_to_2d(target_label) 

1675 target_label.stop_gradient = True

1677 # 3. Mining hard examples 

1678 actual_shape = nn.slice(conf_shape, axes=[0], starts=[0], ends=[2]) 

1679 actual_shape.stop_gradient = True

1681 # actual_shape in runtime.

1683 shape=(-1, 0), 

1684 actual_shape=actual_shape) 

1685 conf_loss.stop_gradient = True 

1686 neg_indices = helper.create_variable_for_type_inference(dtype='int32') 

1687 dtype = matched_indices.dtype

1689 dtype=dtype)

1691 inputs={ 

1692 'ClsLoss': conf_loss, 

1693 'LocLoss': None, 

1694 'MatchIndices': matched_indices, 

1695 'MatchDist': matched_dist, 

1696 }, 

1697 outputs={ 

1698 'NegIndices': neg_indices, 

1699 'UpdatedMatchIndices': updated_matched_indices 

1700 }, 

1701 attrs={

1709 # 4.1. Encoded bbox according to the prior boxes.

1711 prior_box_var=prior_box_var, 

1712 target_box=gt_box, 

1713 code_type='encode_center_size')

1716 encoded_bbox, updated_matched_indices, mismatch_value=background_label) 

1717 # 4.3. Assign classification targets 

1718 target_label, target_conf_weight = target_assign( 

1719 gt_label, 

1720 updated_matched_indices, 

1721 negative_indices=neg_indices, 

1722 mismatch_value=background_label) 

1723 

1724 # 5. Compute loss. 

1725 # 5.1 Compute confidence loss. 

1726 target_label = __reshape_to_2d(target_label) 

1727 target_label = tensor.cast(x=target_label, dtype='int64') 

1728 

1729 conf_loss = softmax_with_cross_entropy(confidence, target_label) 

1730 target_conf_weight = __reshape_to_2d(target_conf_weight) 

1731 conf_loss = conf_loss * target_conf_weight 

1732 

1733 # the target_label and target_conf_weight do not have gradient. 

1734 target_label.stop_gradient = True 

1735 target_conf_weight.stop_gradient = True 

1736 

1737 # 5.2 Compute regression loss. 

1738 location = __reshape_to_2d(location) 

1739 target_bbox = __reshape_to_2d(target_bbox) 

1740 

1741 loc_loss = nn.smooth_l1(location, target_bbox) 

1742 target_loc_weight = __reshape_to_2d(target_loc_weight) 

1743 loc_loss = loc_loss * target_loc_weight 

1744 

1745 # the target_bbox and target_loc_weight do not have gradient. 

1746 target_bbox.stop_gradient = True 

1747 target_loc_weight.stop_gradient = True 

1748 

1749 # 5.3 Compute overall weighted loss. 

1750 loss = conf_loss_weight * conf_loss + loc_loss_weight * loc_loss 

1751 # reshape to [N, Np], N is the batch size and Np is the prior box number. 

1752 # shape=(-1, 0) is set for compile-time, the correct shape is set by 

1753 # actual_shape in runtime. 

1754 loss = nn.reshape(x=loss, shape=(-1, 0), actual_shape=actual_shape) 

1755 loss = nn.reduce_sum(loss, dim=1, keep_dim=True) 

1756 if normalize: 

1757 normalizer = nn.reduce_sum(target_loc_weight) 

1758 loss = loss / normalizer 

1759 

1760 return loss

2867def generate_proposals(scores,

2871 variances,

2873 post_nms_top_n=1000, 

2874 nms_thresh=0.5, 

2875 min_size=0.1,

2877 return_rois_num=False,

2879 """ 

2880 

2881 **Generate proposal Faster-RCNN** 

2882 

2883 This operation proposes RoIs according to each box with their 

2884 probability to be a foreground object and  

2885 the box can be calculated by anchors. Bbox_deltais and scores 

2886 to be an object are the output of RPN. Final proposals 

2887 could be used to train detection net. 

2888 

2889 For generating proposals, this operation performs following steps: 

2890 

2891 1. Transposes and resizes scores and bbox_deltas in size of 

2892 (H*W*A, 1) and (H*W*A, 4)

2907 image information for N batch. Height and width are the input sizes  

2908 and scale is the ratio of network input size and original size.  

2909 The data type can be float32 or float64.

2916 (xcenter, ycenter, w, h) format. The data type must be float32. 

2917 pre_nms_top_n(float): Number of total bboxes to be kept per 

2918 image before NMS. The data type must be float32. `6000` by default. 

2919 post_nms_top_n(float): Number of total bboxes to be kept per 

2920 image after NMS. The data type must be float32. `1000` by default. 

2921 nms_thresh(float): Threshold in NMS. The data type must be float32. `0.5` by default. 

2922 min_size(float): Remove predicted boxes with either height or 

2923 width < min_size. The data type must be float32. `0.1` by default. 

2924 eta(float): Apply in adaptive NMS, if adaptive `threshold > 0.5`, 

2925 `adaptive_threshold = adaptive_threshold * eta` in each iteration. 

2926 return_rois_num(bool): When setting True, it will return a 1D Tensor with shape [N, ] that includes Rois's  

2927 num of each image in one batch. The N is the image's num. For example, the tensor has values [4,5] that represents 

2928 the first image has 4 Rois, the second image has 5 Rois. It only used in rcnn model.  

2929 'False' by default.  

2930 name(str, optional): For detailed information, please refer  

2931 to :ref:`api_guide_Name`. Usually name is no need to set and  

2932 None by default.  

2933 

2934 Returns: 

2935 tuple: 

2936 A tuple with format ``(rpn_rois, rpn_roi_probs)``. 

2937 

2938 - **rpn_rois**: The generated RoIs. 2-D Tensor with shape ``[N, 4]`` while ``N`` is the number of RoIs. The data type is the same as ``scores``. 

2939 - **rpn_roi_probs**: The scores of generated RoIs. 2-D Tensor with shape ``[N, 1]`` while ``N`` is the number of RoIs. The data type is the same as ``scores``. 

2940 

2941 Examples: 

2942 .. code-block:: python 

2943  

2944 import paddle.fluid as fluid 

2945 import paddle 

2946 paddle.enable_static() 

2947 scores = fluid.data(name='scores', shape=[None, 4, 5, 5], dtype='float32') 

2948 bbox_deltas = fluid.data(name='bbox_deltas', shape=[None, 16, 5, 5], dtype='float32') 

2949 im_info = fluid.data(name='im_info', shape=[None, 3], dtype='float32') 

2950 anchors = fluid.data(name='anchors', shape=[None, 5, 4, 4], dtype='float32') 

2951 variances = fluid.data(name='variances', shape=[None, 5, 10, 4], dtype='float32') 

2952 rois, roi_probs = fluid.layers.generate_proposals(scores, bbox_deltas, 

2953 im_info, anchors, variances) 

2954 

2955 """

2957 bbox_deltas=bbox_deltas, 

2958 img_size=im_info[:2], 

2959 anchors=anchors,

2961 pre_nms_top_n=pre_nms_top_n,

2975  

2976 .. code-block:: text 

2977 

2978 xmin = max(min(xmin, im_w - 1), 0) 

2979 ymin = max(min(ymin, im_h - 1), 0)  

2980 xmax = max(min(xmax, im_w - 1), 0) 

2981 ymax = max(min(ymax, im_h - 1), 0) 

2982  

2983 where im_w and im_h are computed from im_info: 

2984

2986

2988 im_w = round(weight / scale) 

2989 

2990 Args:

2992 the last dimension is 4 and data type is float32 or float64.

3006

3033def retinanet_detection_output(bboxes, 

3034 scores, 

3035 anchors, 

3036 im_info, 

3037 score_threshold=0.05, 

3038 nms_top_k=1000, 

3039 keep_top_k=100, 

3040 nms_threshold=0.3, 

3041 nms_eta=1.0): 

3042 """ 

3043 **Detection Output Layer for the detector RetinaNet.** 

3044 

3045 In the detector `RetinaNet <https://arxiv.org/abs/1708.02002>`_ , many  

3046 `FPN <https://arxiv.org/abs/1612.03144>`_ levels output the category 

3047 and location predictions, this OP is to get the detection results by 

3048 performing following steps: 

3049 

3050 1. For each FPN level, decode box predictions according to the anchor 

3051 boxes from at most :attr:`nms_top_k` top-scoring predictions after 

3052 thresholding detector confidence at :attr:`score_threshold`. 

3053 2. Merge top predictions from all levels and apply multi-class non  

3054 maximum suppression (NMS) on them to get the final detections. 

3055 

3056 Args: 

3057 bboxes(List): A list of Tensors from multiple FPN levels represents 

3058 the location prediction for all anchor boxes. Each element is 

3059 a 3-D Tensor with shape :math:`[N, Mi, 4]`, :math:`N` is the 

3060 batch size, :math:`Mi` is the number of bounding boxes from 

3061 :math:`i`-th FPN level and each bounding box has four coordinate 

3062 values and the layout is [xmin, ymin, xmax, ymax]. The data type 

3063 of each element is float32 or float64. 

3064 scores(List): A list of Tensors from multiple FPN levels represents 

3065 the category prediction for all anchor boxes. Each element is a 

3066 3-D Tensor with shape :math:`[N, Mi, C]`, :math:`N` is the batch 

3067 size, :math:`C` is the class number (**excluding background**), 

3068 :math:`Mi` is the number of bounding boxes from :math:`i`-th FPN 

3069 level. The data type of each element is float32 or float64. 

3070 anchors(List): A list of Tensors from multiple FPN levels represents 

3071 the locations of all anchor boxes. Each element is a 2-D Tensor 

3072 with shape :math:`[Mi, 4]`, :math:`Mi` is the number of bounding 

3073 boxes from :math:`i`-th FPN level, and each bounding box has four 

3074 coordinate values and the layout is [xmin, ymin, xmax, ymax]. 

3075 The data type of each element is float32 or float64. 

3076 im_info(Variable): A 2-D Tensor with shape :math:`[N, 3]` represents the size 

3077 information of input images. :math:`N` is the batch size, the size 

3078 information of each image is a 3-vector which are the height and width 

3079 of the network input along with the factor scaling the origin image to 

3080 the network input. The data type of :attr:`im_info` is float32. 

3081 score_threshold(float): Threshold to filter out bounding boxes 

3082 with a confidence score before NMS, default value is set to 0.05. 

3083 nms_top_k(int): Maximum number of detections per FPN layer to be 

3084 kept according to the confidences before NMS, default value is set to 

3085 1000. 

3086 keep_top_k(int): Number of total bounding boxes to be kept per image after 

3087 NMS step. Default value is set to 100, -1 means keeping all bounding 

3088 boxes after NMS step. 

3089 nms_threshold(float): The Intersection-over-Union(IoU) threshold used to  

3090 filter out boxes in NMS. 

3091 nms_eta(float): The parameter for adjusting :attr:`nms_threshold` in NMS. 

3092 Default value is set to 1., which represents the value of 

3093 :attr:`nms_threshold` keep the same in NMS. If :attr:`nms_eta` is set 

3094 to be lower than 1. and the value of :attr:`nms_threshold` is set to 

3095 be higher than 0.5, everytime a bounding box is filtered out, 

3096 the adjustment for :attr:`nms_threshold` like :attr:`nms_threshold` 

3097 = :attr:`nms_threshold` * :attr:`nms_eta` will not be stopped until 

3098 the actual value of :attr:`nms_threshold` is lower than or equal to 

3099 0.5. 

3100 

3101 **Notice**: In some cases where the image sizes are very small, it's possible 

3102 that there is no detection if :attr:`score_threshold` are used at all 

3103 levels. Hence, this OP do not filter out anchors from the highest FPN level 

3104 before NMS. And the last element in :attr:`bboxes`:, :attr:`scores` and 

3105 :attr:`anchors` is required to be from the highest FPN level. 

3106 

3107 Returns: 

3108 Variable(The data type is float32 or float64): 

3109 The detection output is a 1-level LoDTensor with shape :math:`[No, 6]`. 

3110 Each row has six values: [label, confidence, xmin, ymin, xmax, ymax]. 

3111 :math:`No` is the total number of detections in this mini-batch. 

3112 The :math:`i`-th image has `LoD[i + 1] - LoD[i]` detected

3628 To compute FPN level for each roi, the formula is given as follows: 

3629

3643 from.
