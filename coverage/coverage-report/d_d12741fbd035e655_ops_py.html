57def yolo_loss( 

58 x, 

59 gt_box, 

60 gt_label, 

61 anchors, 

62 anchor_mask, 

63 class_num, 

64 ignore_thresh, 

65 downsample_ratio, 

66 gt_score=None, 

67 use_label_smooth=True, 

68 name=None, 

69 scale_x_y=1.0, 

70): 

71 r""" 

72 

73 This operator generates YOLOv3 loss based on given predict result and ground 

74 truth boxes. 

75 

76 The output of previous network is in shape [N, C, H, W], while H and W 

77 should be the same, H and W specify the grid size, each grid point predict 

78 given number bounding boxes, this given number, which following will be represented as S, 

79 is specified by the number of anchor clusters in each scale. In the second dimension(the channel 

80 dimension), C should be equal to S * (class_num + 5), class_num is the object 

81 category number of source dataset(such as 80 in coco dataset), so in the 

82 second(channel) dimension, apart from 4 box location coordinates x, y, w, h, 

83 also includes confidence score of the box and class one-hot key of each anchor box. 

84 

85 Assume the 4 location coordinates are :math:`t_x, t_y, t_w, t_h`, the box predictions 

86 should be as follows: 

87 

88 $$ 

89 b_x = \\sigma(t_x) + c_x 

90 $$ 

91 $$ 

92 b_y = \\sigma(t_y) + c_y 

93 $$ 

94 $$ 

95 b_w = p_w e^{t_w} 

96 $$ 

97 $$ 

98 b_h = p_h e^{t_h} 

99 $$ 

100 

101 In the equation above, :math:`c_x, c_y` is the left top corner of current grid 

102 and :math:`p_w, p_h` is specified by anchors. 

103 

104 As for confidence score, it is the logistic regression value of IoU between 

105 anchor boxes and ground truth boxes, the score of the anchor box which has 

106 the max IoU should be 1, and if the anchor box has IoU bigger than ignore 

107 thresh, the confidence score loss of this anchor box will be ignored. 

108 

109 Therefore, the YOLOv3 loss consists of three major parts: box location loss, 

110 objectness loss and classification loss. The L1 loss is used for 

111 box coordinates (w, h), sigmoid cross entropy loss is used for box 

112 coordinates (x, y), objectness loss and classification loss. 

113 

114 Each groud truth box finds a best matching anchor box in all anchors. 

115 Prediction of this anchor box will incur all three parts of losses, and 

116 prediction of anchor boxes with no GT box matched will only incur objectness 

117 loss. 

118 

119 In order to trade off box coordinate losses between big boxes and small 

120 boxes, box coordinate losses will be mutiplied by scale weight, which is 

121 calculated as follows. 

122 

123 $$ 

124 weight_{box} = 2.0 - t_w * t_h 

125 $$ 

126 

127 Final loss will be represented as follows. 

128 

129 $$ 

130 loss = (loss_{xy} + loss_{wh}) * weight_{box} + loss_{conf} + loss_{class} 

131 $$ 

132 

133 While :attr:`use_label_smooth` is set to be :attr:`True`, the classification 

134 target will be smoothed when calculating classification loss, target of 

135 positive samples will be smoothed to :math:`1.0 - 1.0 / class\_num` and target of 

136 negetive samples will be smoothed to :math:`1.0 / class\_num`. 

137 

138 While :attr:`gt_score` is given, which means the mixup score of ground truth 

139 boxes, all losses incured by a ground truth box will be multiplied by its 

140 mixup score. 

141 

142 Args: 

143 x (Tensor): The input tensor of YOLOv3 loss operator, This is a 4-D 

144 tensor with shape of [N, C, H, W]. H and W should be same, 

145 and the second dimension(C) stores box locations, confidence 

146 score and classification one-hot keys of each anchor box. 

147 The data type is float32 or float64. 

148 gt_box (Tensor): groud truth boxes, should be in shape of [N, B, 4], 

149 in the third dimension, x, y, w, h should be stored. 

150 x,y is the center coordinate of boxes, w, h are the 

151 width and height, x, y, w, h should be divided by 

152 input image height to scale to [0, 1]. 

153 N is the batch number and B is the max box number in 

154 an image.The data type is float32 or float64. 

155 gt_label (Tensor): class id of ground truth boxes, should be in shape 

156 of [N, B].The data type is int32. 

157 anchors (list|tuple): The anchor width and height, it will be parsed 

158 pair by pair. 

159 anchor_mask (list|tuple): The mask index of anchors used in current 

160 YOLOv3 loss calculation. 

161 class_num (int): The number of classes. 

162 ignore_thresh (float): The ignore threshold to ignore confidence loss. 

163 downsample_ratio (int): The downsample ratio from network input to YOLOv3 

164 loss input, so 32, 16, 8 should be set for the 

165 first, second, and thrid YOLOv3 loss operators. 

166 name (string): The default value is None. Normally there is no need 

167 for user to set this property. For more information, 

168 please refer to :ref:`api_guide_Name` 

169 gt_score (Tensor): mixup score of ground truth boxes, should be in shape 

170 of [N, B]. Default None. 

171 use_label_smooth (bool): Whether to use label smooth. Default True. 

172 scale_x_y (float): Scale the center point of decoded bounding box. 

173 Default 1.0 

174 

175 Returns: 

176 Tensor: A 1-D tensor with shape [N], the value of yolov3 loss 

177 

178 Examples: 

179 .. code-block:: python 

180 

181 import paddle 

182 

183 x = paddle.rand([2, 14, 8, 8]).astype('float32') 

184 gt_box = paddle.rand([2, 10, 4]).astype('float32') 

185 gt_label = paddle.rand([2, 10]).astype('int32') 

186 

187 

188 loss = paddle.vision.ops.yolo_loss(x, 

189 gt_box=gt_box, 

190 gt_label=gt_label, 

191 anchors=[10, 13, 16, 30], 

192 anchor_mask=[0, 1], 

193 class_num=2, 

194 ignore_thresh=0.7, 

195 downsample_ratio=8, 

196 use_label_smooth=True, 

197 scale_x_y=1.) 

198 """ 

199 

200 if in_dygraph_mode(): 

201 loss, _, _ = _C_ops.yolov3_loss( 

202 x, 

203 gt_box, 

204 gt_label, 

205 gt_score, 

206 anchors, 

207 anchor_mask, 

208 class_num, 

209 ignore_thresh, 

210 downsample_ratio, 

211 use_label_smooth, 

212 scale_x_y, 

213 ) 

214 return loss

216 if _non_static_mode(): 

217 loss, _, _ = _legacy_C_ops.yolov3_loss( 

218 x, 

219 gt_box, 

220 gt_label, 

221 gt_score, 

222 'anchors', 

223 anchors, 

224 'anchor_mask', 

225 anchor_mask, 

226 'class_num', 

227 class_num, 

228 'ignore_thresh', 

229 ignore_thresh, 

230 'downsample_ratio', 

231 downsample_ratio, 

232 'use_label_smooth', 

233 use_label_smooth, 

234 'scale_x_y', 

235 scale_x_y, 

236 ) 

237 return loss

272 "scale_x_y": scale_x_y, 

273 } 

274 

275 helper.append_op( 

276 type='yolov3_loss', 

277 inputs=inputs, 

278 outputs={ 

279 'Loss': loss, 

280 'ObjectnessMask': objectness_mask,

282 }, 

283 attrs=attrs, 

284 ) 

285 return loss

288def yolo_box( 

289 x, 

290 img_size, 

291 anchors, 

292 class_num, 

293 conf_thresh, 

294 downsample_ratio, 

295 clip_bbox=True, 

296 name=None, 

297 scale_x_y=1.0, 

298 iou_aware=False, 

299 iou_aware_factor=0.5, 

300): 

301 r""" 

302 

303 This operator generates YOLO detection boxes from output of YOLOv3 network. 

304  

305 The output of previous network is in shape [N, C, H, W], while H and W 

306 should be the same, H and W specify the grid size, each grid point predict  

307 given number boxes, this given number, which following will be represented as S, 

308 is specified by the number of anchors. In the second dimension(the channel 

309 dimension), C should be equal to S * (5 + class_num) if :attr:`iou_aware` is false, 

310 otherwise C should be equal to S * (6 + class_num). class_num is the object 

311 category number of source dataset(such as 80 in coco dataset), so the  

312 second(channel) dimension, apart from 4 box location coordinates x, y, w, h,  

313 also includes confidence score of the box and class one-hot key of each anchor  

314 box. 

315 

316 Assume the 4 location coordinates are :math:`t_x, t_y, t_w, t_h`, the box  

317 predictions should be as follows: 

318 

319 $$ 

320 b_x = \\sigma(t_x) + c_x 

321 $$ 

322 $$ 

323 b_y = \\sigma(t_y) + c_y 

324 $$ 

325 $$ 

326 b_w = p_w e^{t_w} 

327 $$ 

328 $$ 

329 b_h = p_h e^{t_h} 

330 $$ 

331 

332 in the equation above, :math:`c_x, c_y` is the left top corner of current grid 

333 and :math:`p_w, p_h` is specified by anchors. 

334 

335 The logistic regression value of the 5th channel of each anchor prediction boxes 

336 represents the confidence score of each prediction box, and the logistic 

337 regression value of the last :attr:`class_num` channels of each anchor prediction  

338 boxes represents the classifcation scores. Boxes with confidence scores less than 

339 :attr:`conf_thresh` should be ignored, and box final scores is the product of  

340 confidence scores and classification scores. 

341 

342 $$ 

343 score_{pred} = score_{conf} * score_{class} 

344 $$ 

345 

346 where the confidence scores follow the formula bellow 

347 

348 .. math:: 

349 

350 score_{conf} = \begin{case} 

351 obj, \text{if } iou_aware == flase \\ 

352 obj^{1 - iou_aware_factor} * iou^{iou_aware_factor}, \text{otherwise} 

353 \end{case} 

354 

355 Args: 

356 x (Tensor): The input tensor of YoloBox operator is a 4-D tensor with 

357 shape of [N, C, H, W]. The second dimension(C) stores box 

358 locations, confidence score and classification one-hot keys 

359 of each anchor box. Generally, X should be the output of 

360 YOLOv3 network. The data type is float32 or float64.  

361 img_size (Tensor): The image size tensor of YoloBox operator, This is a 

362 2-D tensor with shape of [N, 2]. This tensor holds 

363 height and width of each input image used for resizing 

364 output box in input image scale. The data type is int32.  

365 anchors (list|tuple): The anchor width and height, it will be parsed pair 

366 by pair. 

367 class_num (int): The number of classes. 

368 conf_thresh (float): The confidence scores threshold of detection boxes. 

369 Boxes with confidence scores under threshold should 

370 be ignored. 

371 downsample_ratio (int): The downsample ratio from network input to 

372 :attr:`yolo_box` operator input, so 32, 16, 8 

373 should be set for the first, second, and thrid 

374 :attr:`yolo_box` layer. 

375 clip_bbox (bool): Whether clip output bonding box in :attr:`img_size` 

376 boundary. Default true. 

377 scale_x_y (float): Scale the center point of decoded bounding box. 

378 Default 1.0 

379 name (string): The default value is None. Normally there is no need  

380 for user to set this property. For more information,  

381 please refer to :ref:`api_guide_Name` 

382 iou_aware (bool): Whether use iou aware. Default false 

383 iou_aware_factor (float): iou aware factor. Default 0.5 

384 

385 Returns: 

386 Tensor: A 3-D tensor with shape [N, M, 4], the coordinates of boxes, 

387 and a 3-D tensor with shape [N, M, :attr:`class_num`], the classification  

388 scores of boxes. 

389 

390 Examples: 

391 

392 .. code-block:: python 

393 

394 import paddle 

395 

396 x = paddle.rand([2, 14, 8, 8]).astype('float32') 

397 img_size = paddle.ones((2, 2)).astype('int32') 

398 

399 boxes, scores = paddle.vision.ops.yolo_box(x, 

400 img_size=img_size, 

401 anchors=[10, 13, 16, 30], 

402 class_num=2, 

403 conf_thresh=0.01, 

404 downsample_ratio=8, 

405 clip_bbox=True, 

406 scale_x_y=1.) 

407 """ 

408 if in_dygraph_mode(): 

409 boxes, scores = _C_ops.yolo_box( 

410 x, 

411 img_size, 

412 anchors, 

413 class_num, 

414 conf_thresh, 

415 downsample_ratio, 

416 clip_bbox, 

417 scale_x_y, 

418 iou_aware, 

419 iou_aware_factor, 

420 ) 

421 return boxes, scores 

422 

423 if _non_static_mode(): 

424 boxes, scores = _legacy_C_ops.yolo_box( 

425 x, 

426 img_size, 

427 'anchors', 

428 anchors, 

429 'class_num', 

430 class_num, 

431 'conf_thresh', 

432 conf_thresh, 

433 'downsample_ratio', 

434 downsample_ratio, 

435 'clip_bbox', 

436 clip_bbox, 

437 'scale_x_y', 

438 scale_x_y, 

439 'iou_aware', 

440 iou_aware, 

441 'iou_aware_factor', 

442 iou_aware_factor, 

443 ) 

444 return boxes, scores 

445 

446 helper = LayerHelper('yolo_box', **locals()) 

447 

448 check_variable_and_dtype(x, 'x', ['float32', 'float64'], 'yolo_box') 

449 check_variable_and_dtype(img_size, 'img_size', 'int32', 'yolo_box') 

450 check_type(anchors, 'anchors', (list, tuple), 'yolo_box') 

451 check_type(conf_thresh, 'conf_thresh', float, 'yolo_box') 

452 

453 boxes = helper.create_variable_for_type_inference(dtype=x.dtype) 

454 scores = helper.create_variable_for_type_inference(dtype=x.dtype) 

455 

456 attrs = { 

457 "anchors": anchors, 

458 "class_num": class_num, 

459 "conf_thresh": conf_thresh, 

460 "downsample_ratio": downsample_ratio, 

461 "clip_bbox": clip_bbox, 

462 "scale_x_y": scale_x_y, 

463 "iou_aware": iou_aware, 

464 "iou_aware_factor": iou_aware_factor, 

465 } 

466 

467 helper.append_op( 

468 type='yolo_box', 

469 inputs={ 

470 "X": x, 

471 "ImgSize": img_size, 

472 }, 

473 outputs={ 

474 'Boxes': boxes, 

475 'Scores': scores, 

476 }, 

477 attrs=attrs, 

478 ) 

479 return boxes, scores

482def prior_box( 

483 input, 

484 image, 

485 min_sizes, 

486 max_sizes=None, 

487 aspect_ratios=[1.0], 

488 variance=[0.1, 0.1, 0.2, 0.2], 

489 flip=False, 

490 clip=False, 

491 steps=[0.0, 0.0], 

492 offset=0.5, 

493 min_max_aspect_ratios_order=False, 

494 name=None, 

495): 

496 r""" 

497 

498 This op generates prior boxes for SSD(Single Shot MultiBox Detector) algorithm. 

499 

500 Each position of the input produce N prior boxes, N is determined by 

501 the count of min_sizes, max_sizes and aspect_ratios, The size of the 

502 box is in range(min_size, max_size) interval, which is generated in 

503 sequence according to the aspect_ratios. 

504 

505 Args: 

506 input (Tensor): 4-D tensor(NCHW), the data type should be float32 or float64. 

507 image (Tensor): 4-D tensor(NCHW), the input image data of PriorBoxOp, 

508 the data type should be float32 or float64. 

509 min_sizes (list|tuple|float): the min sizes of generated prior boxes. 

510 max_sizes (list|tuple|None, optional): the max sizes of generated prior boxes. 

511 Default: None. 

512 aspect_ratios (list|tuple|float, optional): the aspect ratios of generated 

513 prior boxes. Default: [1.]. 

514 variance (list|tuple, optional): the variances to be encoded in prior boxes. 

515 Default:[0.1, 0.1, 0.2, 0.2]. 

516 flip (bool): Whether to flip aspect ratios. Default:False. 

517 clip (bool): Whether to clip out-of-boundary boxes. Default: False. 

518 steps (list|tuple, optional): Prior boxes steps across width and height, If 

519 steps[0] equals to 0.0 or steps[1] equals to 0.0, the prior boxes steps across 

520 height or weight of the input will be automatically calculated. 

521 Default: [0., 0.] 

522 offset (float, optional)): Prior boxes center offset. Default: 0.5 

523 min_max_aspect_ratios_order (bool, optional): If set True, the output prior box is 

524 in order of [min, max, aspect_ratios], which is consistent with 

525 Caffe. Please note, this order affects the weights order of 

526 convolution layer followed by and does not affect the final 

527 detection results. Default: False. 

528 name (str, optional): The default value is None. Normally there is no need for 

529 user to set this property. For more information, please refer to :ref:`api_guide_Name` 

530 

531 Returns: 

532 Tensor: the output prior boxes and the expanded variances of PriorBox. 

533 The prior boxes is a 4-D tensor, the layout is [H, W, num_priors, 4], 

534 num_priors is the total box count of each position of input. 

535 The expanded variances is a 4-D tensor, same shape as the prior boxes. 

536 

537 Examples: 

538 .. code-block:: python 

539 

540 import paddle 

541 

542 input = paddle.rand((1, 3, 6, 9), dtype=paddle.float32) 

543 image = paddle.rand((1, 3, 9, 12), dtype=paddle.float32) 

544 

545 box, var = paddle.vision.ops.prior_box( 

546 input=input, 

547 image=image, 

548 min_sizes=[2.0, 4.0], 

549 clip=True, 

550 flip=True) 

551 

552 """ 

553 helper = LayerHelper("prior_box", **locals()) 

554 dtype = helper.input_dtype() 

555 check_variable_and_dtype( 

556 input, 'input', ['uint8', 'int8', 'float32', 'float64'], 'prior_box' 

557 ) 

558 

559 def _is_list_or_tuple_(data): 

560 return isinstance(data, list) or isinstance(data, tuple) 

561 

562 if not _is_list_or_tuple_(min_sizes): 

563 min_sizes = [min_sizes] 

564 if not _is_list_or_tuple_(aspect_ratios): 

565 aspect_ratios = [aspect_ratios] 

566 if not _is_list_or_tuple_(steps): 

567 steps = [steps] 

568 if not len(steps) == 2: 

569 raise ValueError('steps should be (step_w, step_h)') 

570 

571 min_sizes = list(map(float, min_sizes)) 

572 aspect_ratios = list(map(float, aspect_ratios)) 

573 steps = list(map(float, steps)) 

574 

575 cur_max_sizes = None 

576 if max_sizes is not None and len(max_sizes) > 0 and max_sizes[0] > 0: 

577 if not _is_list_or_tuple_(max_sizes): 

578 max_sizes = [max_sizes] 

579 cur_max_sizes = max_sizes 

580 

581 if in_dygraph_mode(): 

582 step_w, step_h = steps 

583 if max_sizes == None: 

584 max_sizes = [] 

585 box, var = _C_ops.prior_box( 

586 input, 

587 image, 

588 min_sizes, 

589 aspect_ratios, 

590 variance, 

591 max_sizes, 

592 flip, 

593 clip, 

594 step_w, 

595 step_h, 

596 offset, 

597 min_max_aspect_ratios_order, 

598 ) 

599 return box, var

602 attrs = (

621 )

626 else:

629 'aspect_ratios': aspect_ratios,
